# 第1章 イントロダクション

このガイドでは、実践的な例を通じて Rust での Web API 開発を学びます。

## なぜ Rust で Web API を作るのか？

Rust は以下の特徴を持つプログラミング言語です：

1. **メモリ安全**：[[ownership]]システムにより、ビルド時に多くのバグが検出されます
2. **高速**：C/C++ に匹敵する実行速度を持ちながら、安全性は圧倒的に高い
3. **並行処理に強い**：[[async-await]] により、少ないリソースで大量の同時接続を処理可能
4. **ゼロコスト抽象化**：高度な型システムを使っても、オーバーヘッドがない

## 実装予定のサンプル

このガイドでは、シンプルな**ユーザー管理 API** を実装します。以下のエンドポイントを提供します：

:::message
エンドポイント一覧：

- `GET /users` - ユーザー一覧取得
- `GET /users/{id}` - ユーザー詳細取得
- `POST /users` - ユーザー登録
- `PUT /users/{id}` - ユーザー更新
- `DELETE /users/{id}` - ユーザー削除
:::

## 前提知識

このガイドを進めるには、以下の知識があると良いです：

- Rust の基本構文（変数、関数、制御フロー）
- HTTP の基本概念（GET、POST など）
- [[async-await]] の基本的な理解

Rust 自体が初めての場合は、[公式ブック（The Book）](https://doc.rust-lang.org/book/)で基礎を学ぶことをお勧めします。

## 環境要件

- Rust 1.70 以上
- Cargo（Rust パッケージマネージャ）
- テキストエディタまたは IDE（VS Code + Rust Analyzer 推奨）
