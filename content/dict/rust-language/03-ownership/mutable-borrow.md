可変借用は、[[mutable-reference]]を作って値を「書き換えられる状態で借りる」[[borrow]]です。借りている間も[[ownership]]は所有者に残ったままですが、共有借用と違って**排他的**で、1つの値に対する可変借用は同時に1つしか存在できません。

```rust playground
fn main() {
    let mut score = 80;
    let r1 = &mut score;
    *r1 += 10; // ここがr1の最後の使用（ここで可変借用が終わる）
    let r2 = &mut score; // r1の借用が終わっているので、次の可変借用を作れる
    *r2 += 5;
    println!("最終スコア: {score}点");
}
```

このように「同時に」1つであればよく、前の借用が終われば順番に何度でも可変借用できます（借用がいつ終わるかは[[borrow]]を参照）。

## 可変借用中に禁止される操作

可変借用が有効な間は、次の操作がすべてコンパイルエラーになります。

| 禁止される操作 | エラー |
| --- | --- |
| ほかの可変借用を作る | E0499 |
| 共有借用を作る | E0502 |
| 所有者を通じて値を使う | E0503 |

:::message{tip}
メソッド呼び出しでは可変借用が自動で行われます。たとえば`items.push(value)`は`(&mut items).push(value)`とほぼ同等で、`&mut`を書かなくても`items`の可変借用が発生しています。
:::

## 補足

:::details[同時に2つは作れない（E0499）]
可変借用が2つ同時に存在すると、同じ値への書き換えが競合し、データ競合の原因になります。Rustはこれをコンパイル時に禁止します。

<!-- rustc: expect E0499 -->
```rust playground
fn main() {
    let mut score = 80;
    let r1 = &mut score;
    let r2 = &mut score; // エラー: E0499（r1の借用が終わる前に2つ目の可変借用）
    println!("{r1}点と{r2}点");
}
```
:::
