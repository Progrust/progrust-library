**NLL**（Non-Lexical Lifetimes、非レキシカルライフタイム）は、[[borrow]]がいつまで続くかを、字句上のスコープ（レキシカルスコープ）の終わりまでではなく、制御フロー上で[[reference]]が最後に使用される地点までとして判定する借用チェッカーの解析方式です。<!-- TODO: [[scope]] 作成後にリンク --><!-- TODO: [[borrow-checker]] 作成後にリンク -->

借用は最後の使用とともに終了するため、参照を保持する[[variable]]がまだスコープ内にあっても、その後に同じ値への排他借用を開始するコードが受け入れられます。

```rust playground
fn main() {
    let mut scores = vec![80, 95, 70];
    let first = &scores[0]; // 共有借用の開始
    println!("最初の点数: {first}"); // 「first」の最後の使用。共有借用はここで終わる
    scores.push(60); // 共有借用は終わっているので排他借用できる
    println!("全科目: {scores:?}");
}
```

## 補足

:::details[NLL以前の挙動]
NLL導入以前の借用チェッカーは、借用が「参照を保持する変数のスコープ終端」まで続くとみなしていました（レキシカルライフタイム）。そのため上のコード例は、`first`を最後に使用したあとであっても`scores.push(60)`の排他借用がエラーになっていました。実際にはもう使われない参照のために書き換えがブロックされるこの不便さを解消したのがNLLです。

なお、参照を保持する値が、デストラクタでその参照を使用しうる形で`Drop`トレイトを実装している場合は<!-- TODO: [[trait]] 作成後にリンク -->、NLLのもとでもスコープ終端でのデストラクタ実行が「使用」とみなされ、借用はそこまで延長されます。この規則を含む借用の有効期間の全体像は[[borrow]]の「借用はいつまで続くか」を参照してください。
:::

:::details[導入の経緯とバージョン]
NLLは[RFC 2094](https://rust-lang.github.io/rfcs/2094-nll.html)で提案されました。従来のようにソースコードの字句構造（スコープのネスト）からライフタイムを決めるのではなく<!-- TODO: [[lifetime]] 作成後にリンク -->、コンパイラの中間表現（MIR）の制御フローグラフ上で借用の有効範囲を解析することで、「実際に使われている範囲」に基づいた検査を実現しています。

| バージョン | 内容 |
| --- | --- |
| Rust 1.31（2018年12月） | Rust 2018エディション向けにNLLを安定化 |
| Rust 1.36（2019年7月） | Rust 2015エディションでもNLLを有効化 |

現在のRustでは、エディションに関わらずNLLに基づいて借用検査が行われます。
:::
