参照は、[[ownership]]を移動させずに値を指し示す値です。[[variable]]`x`の前に`&`を付けた`&x`で作成でき、この「参照を作って値を借りる」行為を[[borrow]]と呼びます。参照を渡せば、値そのものを渡す（[[move]]する）ことなく、渡した先で値を利用できます。

```rust playground
fn count_chars(text: &str) -> usize {
    text.chars().count() // 参照経由で値を読める
}

fn main() {
    let message = String::from("こんにちは");
    let len = count_chars(&message); // 参照を渡す（所有権はムーブしない）
    println!("「{message}」は{len}文字"); // 所有権が残っているので引き続き使える
}
```

## 2種類の参照

| 種類 | 型 | できること | 同時に存在できる数 |
| --- | --- | --- | --- |
| 共有参照（不変参照） | `&T` | 読み取り | いくつでも |
| 可変参照 | `&mut T` | 読み取り・書き換え | 1つだけ |

この「共有はいくつでも、可変はただ1つ」という制約の詳細は[[borrow]]の借用規則を参照してください。

## 可変参照

可変参照`&mut T`は、参照先の値を書き換えられる参照です。`mut`付きで宣言された変数に対して`&mut x`で作成し、書き換えは`*`（デリファレンス）を通して行います。可変参照を作る行為は可変借用と呼ばれ、その借用が有効な間はほかの参照と同時には使えない**排他性**が保証されます（詳細は[[borrow]]）。

```rust playground
fn deposit(balance: &mut i32, amount: i32) {
    *balance += amount; // 参照先の値を書き換える
}

fn main() {
    let mut balance = 1000; // 変数を可変借用するには、その変数自体がmutであることが必要
    deposit(&mut balance, 500);
    println!("入金後の残高: {balance}円");
}
```

## 補足

:::details[参照は無効な値を指せない]
C/C++のポインタと違い、Rustの参照は「常に有効な値を指している」ことがコンパイル時に保証されます（`unsafe`を使わないコードの場合）。nullは存在せず、破棄された値を指し続ける参照（ダングリング参照）を作るコードはコンパイルエラーになります。

<!-- rustc: expect E0597 -->
```rust playground
fn main() {
    let r;
    {
        let s = String::from("一時的な値");
        r = &s; // エラー: E0597（sは次の行で破棄されるのに、参照が生き残ってしまう）
    } // ここで s が破棄される
    println!("{r}");
}
```

この保証を支える「参照がどこまで有効か」という概念はライフタイムと呼ばれます。<!-- TODO: [[lifetime]] 作成後にリンク -->
:::

:::details[mutでない変数からは可変参照を作れない]
可変参照は「書き換えてよい」という許可を借りるものなので、そもそも書き換えを許していない（`mut`なしで宣言された）変数からは作れません。

<!-- rustc: expect E0596 -->
```rust playground
fn main() {
    let balance = 1000; // mutが付いていない
    let r = &mut balance; // エラー: E0596（不変の変数は可変借用できない）
    *r += 500;
}
```
:::
