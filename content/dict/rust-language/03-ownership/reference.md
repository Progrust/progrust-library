参照は、[[ownership]]を移動させずに値を指し示す値です。[[variable]]`x`の前に`&`を付けた`&x`で作成でき、この「参照を作って値を借りる」行為を[[borrow]]と呼びます。参照を渡せば、値そのものを渡す（[[move]]する）ことなく、渡した先で値を利用できます。

```rust playground
fn count_chars(text: &str) -> usize {
    text.chars().count() // 参照経由で値を読める
}

fn main() {
    let message = String::from("こんにちは");
    let len = count_chars(&message); // 参照を渡す（所有権はムーブしない）
    println!("「{message}」は{len}文字"); // 所有権が残っているので引き続き使える
}
```

## 2種類の参照

| 種類 | 型 | できること | 同時に存在できる数 |
| --- | --- | --- | --- |
| 共有参照（不変参照） | `&T` | 読み取り | いくつでも |
| [[mutable-reference]] | `&mut T` | 読み取り・書き換え | 1つだけ |

この「共有はいくつでも、可変はただ1つ」という制約の詳細は[[borrow]]の借用規則を参照してください。

## 補足

:::details[参照は無効な値を指せない]
C/C++のポインタと違い、Rustの参照は「常に有効な値を指している」ことがコンパイル時に保証されます（`unsafe`を使わないコードの場合）。nullは存在せず、破棄された値を指し続ける参照（ダングリング参照）を作るコードはコンパイルエラーになります。

<!-- rustc: expect E0597 -->
```rust playground
fn main() {
    let r;
    {
        let s = String::from("一時的な値");
        r = &s; // エラー: E0597（sは次の行で破棄されるのに、参照が生き残ってしまう）
    } // ここで s が破棄される
    println!("{r}");
}
```
