借用（Borrowing）は、[[reference]]を作って値を「借りる」行為です。[[ownership]]は元の所有者に残ったままなので、[[move]]と違い、借りた側が使い終わったあとも所有者は値を使い続けられます。共有参照`&T`を作ることを共有借用（不変借用）、[[mutable-reference]]`&mut T`を作ることを[[mutable-borrow]]と呼びます。

コンパイラはすべての借用が次の**借用規則**を守ることをコンパイル時に検査し、違反はコンパイルエラーにします。

1. ある値に対して、**任意の個数の共有借用**または**ただ1つの可変借用**の、どちらか一方しか同時に存在できません
2. 参照は常に有効な値を指していなければなりません

```rust playground
fn main() {
    let mut balance = 1000;
    let r1 = &balance;
    let r2 = &balance;
    println!("残高の確認: {r1}円と{r2}円"); // 共有借用は同時にいくつでも作れる

    let r3 = &mut balance; // r1・r2はもう使われないため、可変借用を作れる
    *r3 += 500;
    println!("入金後の残高: {balance}円");
}
```

## 借用はいつまで続くか

借用は、参照が作られてからその参照が**最後に使われる地点まで**続きます（スコープ<!-- TODO: [[scope]] 作成後にリンク -->の終わりまでではありません）。上の例で`r3`を作れるのは、`r1`・`r2`の借用が`println!`の行で終わっているためです。ただし、参照を保持している値が`Drop`トレイトを実装している場合は、スコープ終端でのデストラクタ実行も「使用」とみなされ、借用はそこまで延長されます。この検査を行うコンパイラの仕組みは借用チェッカーと呼ばれます。<!-- TODO: [[borrow-checker]] 作成後にリンク -->

## 補足

:::details[借用規則に違反する例（E0502）]
共有借用が生きている間に可変借用が必要な操作をすると、借用規則違反でコンパイルエラーになります。もしこのコードが許されると、`push`による内部メモリの再確保で`first`が無効なメモリを指す恐れがあります。借用規則はこうした危険をコンパイル時に排除するためのものです。

<!-- rustc: expect E0502 -->
```rust playground
fn main() {
    let mut items = vec![String::from("りんご")];
    let first = &items[0]; // 共有借用
    items.push(String::from("みかん")); // エラー: E0502（pushはitemsの可変借用を必要とする）
    println!("最初の商品: {first}");
}
```
:::
