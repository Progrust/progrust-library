借用（Borrowing）は、[[reference]]を作って値を「借りる」行為です。[[ownership]]は元の所有者に残ったままなので、[[move]]と違い、借りた側が使い終わったあとも所有者は値を使い続けられます。共有参照`&T`を作ることを共有借用（不変借用）、可変参照`&mut T`を作ることを可変借用と呼びます。

コンパイラはすべての借用が次の**借用規則**を守ることをコンパイル時に検査し、違反はコンパイルエラーにします。

1. ある値に対して、**任意の個数の共有借用**または**ただ1つの可変借用**の、どちらか一方しか同時に存在できません
2. 参照は常に有効な値を指していなければなりません

```rust playground
fn main() {
    let mut balance = 1000;
    let r1 = &balance;
    let r2 = &balance;
    println!("残高の確認: {r1}円と{r2}円"); // 共有借用は同時にいくつでも作れる

    let r3 = &mut balance; // r1・r2はもう使われないため、可変借用を作れる
    *r3 += 500;
    println!("入金後の残高: {balance}円");
}
```

## 借用はいつまで続くか

借用は、参照が作られてからその参照が**最後に使われる地点まで**続きます（スコープ<!-- TODO: [[scope]] 作成後にリンク -->の終わりまでではありません）。上の例で`r3`を作れるのは、`r1`・`r2`の借用が`println!`の行で終わっているためです。ただし、参照を保持している値が`Drop`トレイトを実装している場合は、スコープ終端でのデストラクタ実行も「使用」とみなされ、借用はそこまで延長されます。この検査を行うコンパイラの仕組みは借用チェッカーと呼ばれます。<!-- TODO: [[borrow-checker]] 作成後にリンク -->

## 可変借用の排他性

[[reference]]の可変参照`&mut T`を作る可変借用は、書き換えの競合（データ競合）を防ぐため排他的で、可変借用が有効な間は次の操作がすべてコンパイルエラーになります。

| 禁止される操作 | エラー |
| --- | --- |
| ほかの可変借用を作る | E0499 |
| 共有借用を作る | E0502 |
| 所有者を通じて値を使う | E0503 |

「同時に」1つであればよいため、前の借用が終われば順番に何度でも可変借用できます。

:::message{tip}
メソッド呼び出しでは可変借用が自動で行われます。たとえば`items.push(value)`は`(&mut items).push(value)`とほぼ同等で、`&mut`を書かなくても`items`の可変借用が発生しています。
:::

## 補足

:::details[借用規則に違反する例（E0502）]
共有借用が生きている間に可変借用が必要な操作をすると、借用規則違反でコンパイルエラーになります。もしこのコードが許されると、`push`による内部メモリの再確保で`first`が無効なメモリを指す恐れがあります。借用規則はこうした危険をコンパイル時に排除するためのものです。

<!-- rustc: expect E0502 -->
```rust playground
fn main() {
    let mut items = vec![String::from("りんご")];
    let first = &items[0]; // 共有借用
    items.push(String::from("みかん")); // エラー: E0502（pushはitemsの可変借用を必要とする）
    println!("最初の商品: {first}");
}
```
:::

:::details[可変借用を同時に2つは作れない（E0499）]
可変借用が2つ同時に存在すると、同じ値への書き換えが競合し、データ競合の原因になります。Rustはこれをコンパイル時に禁止します。

<!-- rustc: expect E0499 -->
```rust playground
fn main() {
    let mut score = 80;
    let r1 = &mut score;
    let r2 = &mut score; // エラー: E0499（r1の借用が終わる前に2つ目の可変借用）
    println!("{r1}点と{r2}点");
}
```

前の借用が終わってから借り直すのは問題ありません。

```rust playground
fn main() {
    let mut score = 80;
    let r1 = &mut score;
    *r1 += 10; // ここがr1の最後の使用（ここで可変借用が終わる）
    let r2 = &mut score; // r1の借用が終わっているので、次の可変借用を作れる
    *r2 += 5;
    println!("最終スコア: {score}点");
}
```
:::
