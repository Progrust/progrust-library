# 検索 仕様

## 1. 概要

ヘッダーの検索ボックスと、一覧ページのタグ・キーワード絞込を定義する。両者はビルド時に生成する同一の検索インデックスJSONを利用する。

関連文書:

- 検索ボックス・タグチップの見た目: [`../ui-design/ui-design-spec.md`](../ui-design/ui-design-spec.md)
- インデックス生成の実装方式: [`../architecture.md`](../architecture.md)

## 2. 要求仕様

### 検索インデックス

- **R-1**: ビルド時に全公開コンテンツ（章も含む）のメタ情報をJSONとして出力する。非公開コンテンツは含めない（本番ビルド）。
- **R-2**: インデックスは検索ボックスへの**初回フォーカス時に遅延ロード**する（初期表示に影響させない）。一覧ページの絞込でも同じインデックスを利用する。

### クエリ構文

- **R-3**: キーワードとタグ（`#` + タグ名）で検索できる。
- **R-4**: キーワードはタイトル・概要（description）・タグ名を対象に部分一致で検索する。大文字小文字は区別しない。
- **R-5**: タグ（`#タグ名`）はタグ名の完全一致で検索する。
- **R-6**: 複数キーワード・複数タグはAND検索とする。キーワードとタグは同時に指定可能（すべての条件を満たすものがヒット）。

### 検索ボックス（ヘッダー）

- **R-7**: 検索結果は検索ボックス下部にドロップダウンで表示する。各結果にコンテンツ種別バッジを付ける。
- **R-8**: 結果クリックで対象ページへ遷移する。章がヒットした場合は章詳細ページへ、本（index）がヒットした場合は本トップページへ遷移する。

### 一覧ページの絞込

- **R-9**: 辞書・記事・本の各一覧ページで、タグチップの選択とキーワード入力による絞込ができる。タグはAND、キーワードはR-4と同じ部分一致。絞込はクライアント側で行い、ページ遷移しない。
- **R-10**: 絞込結果の件数（`N / 総数`）を表示し、0件時はその旨のメッセージを表示する。
- **R-11**: 一覧の絞込対象はその一覧の種別のみとする（辞書一覧では辞書のみ。ただしタグ詳細ページは全種別）。

## 3. データ定義

検索インデックスJSON（1エントリ）:

```json
{
  "title": "所有権",
  "description": "Rustのメモリ管理の中核概念。",
  "tags": ["Rust基礎", "所有権"],
  "type": "dict",
  "url": "/dict/ownership"
}
```

- `type`: `"dict" | "article" | "book" | "chapter"`
- 章エントリの`url`は章詳細ページ、本エントリの`url`は本トップページ
- 配信URL・生成方式は [`../architecture.md`](../architecture.md) で定義

## 4. 受入基準

- **AC-1**: ビルド成果物に検索インデックスJSONが含まれ、全公開コンテンツ（章含む）のエントリがあり、非公開コンテンツのエントリがない。（R-1）
- **AC-2**: ページ初期ロードではインデックスJSONがfetchされず、検索ボックスへの初回フォーカスでfetchされる。（R-2）
- **AC-3**: `所有権` で、タイトル・概要・タグ名のいずれかに部分一致する全種別のコンテンツがヒットする。（R-4）
- **AC-4**: `#Rust基礎` でタグ完全一致のコンテンツのみがヒットし、部分一致（例: タグ `Rust`）ではヒットしない。（R-5）
- **AC-5**: `#Rust基礎 所有権` で、タグ完全一致かつキーワード部分一致のANDになる。（R-6）
- **AC-6**: 検索結果ドロップダウンに種別バッジが表示され、章の結果クリックで章詳細ページへ遷移する。（R-7, R-8）
- **AC-7**: 辞書一覧でタグチップを2つ選択すると、両タグを持つ辞書のみに絞り込まれ、件数表示が更新される。（R-9, R-10）

## 5. 未確定事項

- キーワード正規化の範囲（全角/半角・ひらがな/カタカナの同一視）は実装時に決める（初期実装は大文字小文字の同一視のみでよい）。
- 検索ボックスのキーボード操作（矢印キーでの結果選択）を実装するかは実装時に決める。
- ドロップダウンの最大表示件数は実装時に決める。
