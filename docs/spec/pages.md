# ページ 仕様

## 1. 概要

全画面の構成・URL・表示内容・並び順を定義する。

関連文書:

- 見た目（レイアウト・コンポーネントデザイン）: [`../ui-design/ui-design-spec.md`](../ui-design/ui-design-spec.md)
- 検索ボックス・一覧絞込の挙動: [`search.md`](search.md)
- テーマ切替ボタンの挙動: [`theme.md`](theme.md)
- 辞書表示欄・使用辞書一覧・逆リンク・ホバープレビュー: [`wikilink-ui.md`](wikilink-ui.md)
- コンテンツ種別・URLのslug規則: [`content-model.md`](content-model.md)

## 2. 要求仕様

### URL設計

- **R-1**: 画面とURLの対応は以下とする。
  | 画面 | URL |
  | --- | --- |
  | トップ | `/` |
  | 辞書一覧 | `/dict/` |
  | 辞書詳細 | `/dict/[slug]` |
  | 記事一覧 | `/articles/` |
  | 記事詳細 | `/articles/[slug]` |
  | 本一覧 | `/books/` |
  | 本トップ | `/books/[本のslug]` |
  | 章詳細 | `/books/[本のslug]/[章のslug]` |
  | タグ一覧 | `/tags/` |
  | タグ詳細 | `/tags/[タグ名]` |
  | プロフィール | `/profile/` |

### 全ページ共通

- **R-2**: ヘッダー（スクロール固定）: トップページへのロゴリンク + 検索ボックス（[`search.md`](search.md)）+ テーマ切替ボタン（[`theme.md`](theme.md)）の3点のみ。
- **R-3**: フッター: コピーライト表記 + プロフィール / GitHub / X / RSS へのテキストリンク。
- **R-4**: 一覧・新着の並び順は `created_at` の降順。同日の場合はタイトルの昇順でタイブレークする。

### トップページ（`/`）

- **R-5**: 新着一覧を表示する。`created_at` 降順で10件、辞書・記事・本の3タイプ混在（章は含めない。本の新着判定は `index.md` の `created_at`）。各項目にコンテンツ種別バッジを表示する。
- **R-6**: 辞書一覧・記事一覧・本一覧・タグ一覧への遷移リンクを置く。
- **R-7**: プロフィールはトップに置かず、SNS/GitHubアイコンとプロフィールページへのリンクボタンのみ表示する。

### 一覧ページ

- **R-8**: 辞書一覧（`/dict/`）: タイトル+タグのみのカードスタイルのグリッド表示。タグとキーワードで絞込可能（[`search.md`](search.md)）。
- **R-9**: 記事一覧（`/articles/`）: 概要・ヘッダ画像を含むカードスタイルの縦並び。タグとキーワードで絞込可能。
- **R-10**: 本一覧（`/books/`）: 概要・ヘッダ画像を含むカードスタイルの縦並び。タグとキーワードで絞込可能。
- **R-11**: 一覧はページネーションせず全件表示する（絞込はクライアント側で行う）。

### 詳細ページ共通（辞書・記事・章）

- **R-12**: 中央に タイトル / タグ一覧 / 更新日 / 本文 を表示する。タグクリックでタグ詳細ページへ遷移する。
- **R-13**: 左に目次を表示する（モバイルではフローティングボタン押下でポップアップ）。対象は**h2〜h4**（本文先頭のh1はページタイトルと重複するため目次に含めない。脚注ラベルも除外する）。スクロールに応じて現在地の見出しをハイライトする。本の章では、章目次（本の全章リスト）も同時に表示する（本文の目次は開いているページのみ）。
- **R-14**: 右に辞書表示欄（サイドペイン）と使用されている辞書一覧を表示する（仕様は [`wikilink-ui.md`](wikilink-ui.md)。モバイルでの配置も同文書）。
- **R-15**: 辞書詳細では、本文下にその辞書の逆リンク一覧を表示する（[`wikilink-ui.md`](wikilink-ui.md)）。
- **R-16**: 章詳細では、本文下に前の章 / 次の章のナビゲーションを表示する。順序はファイル名連番の昇順（[`content-model.md`](content-model.md) R-7）。非公開の章はスキップして次の公開章へつなぐ。先頭章に「前の章」、最終章に「次の章」は表示しない。

### 本トップページ（`/books/[slug]`）

- **R-17**: 本の概要（`index.md` の本文）と章目次を表示する。章目次は連番昇順、非公開の章は載せない（本番ビルド時）。

### タグページ

- **R-18**: タグ一覧（`/tags/`）: 公開コンテンツで使用されている全タグを表示する。
- **R-19**: タグ詳細（`/tags/[タグ名]`）: そのタグを使用している公開ページの一覧を種別バッジ付きで表示する。章コンテンツも対象に含める。

### プロフィールページ（`/profile/`）

- **R-20**: アバター + 自己紹介 + SNSボタン群 + 「このサイトについて」の3形式（辞書・記事・本）紹介カードを表示する。内容は `profile.html` モック（[`ui-design-spec.md`](../ui-design/ui-design-spec.md)）に従い、`src/pages/profile.astro` に直書きする。

### 404ページ

- **R-21**: サイト共通デザイン（ヘッダー・フッター付き）の404ページを用意し、トップへの導線を置く。

## 4. 受入基準

- **AC-1**: R-1の全URLがビルドで生成される（404含む）。
- **AC-2**: トップの新着に辞書・記事・本が混在して `created_at` 降順で最大10件表示され、章は現れない。（R-5）
- **AC-3**: 全一覧ページが全件表示され、ページ送りUIが存在しない。（R-11）
- **AC-4**: 章詳細で前後章ナビが連番順に機能し、間に非公開章がある場合はスキップされる。（R-16）
- **AC-5**: 本トップの章目次から非公開章が除外されている（本番ビルド）。（R-17）
- **AC-6**: タグ詳細に、そのタグを持つ章コンテンツが表示される。（R-19）
- **AC-7**: 詳細ページの目次にh2〜h4の見出しが階層表示され、クリックで該当見出しへ移動する。（R-13）
- **AC-8**: 存在しないURLへアクセスすると共通デザインの404ページが表示される。（R-21）
