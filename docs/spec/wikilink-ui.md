# wikilink UI 仕様

## 1. 概要

辞書リンク（wikilink）のクリック・ホバー時のクライアント側の振る舞い（ホバープレビュー・サイドペイン・embedパーシャル・使用辞書一覧・逆リンク）を定義する。

分担線:

- wikilink記法のmdast変換・出力HTML/data属性・ビルド時検証の実装: [`../markdown-pipeline/wikilink.md`](../markdown-pipeline/wikilink.md)
- リンク切れ・公開非対称のビルドエラー仕様: [`content-model.md`](content-model.md) R-13〜R-15
- プレビュー小窓・サイドペイン・辞書リンクの見た目: [`../ui-design/ui-design-spec.md`](../ui-design/ui-design-spec.md)
- 本spec: 変換後のHTMLをクライアントJSでどう拡張するか

## 2. 要求仕様

### wikilink記法と変換

- **R-1**: wikilinkは `[[ファイル名]]` でmdファイル名（拡張子なし）を指定する。HTML表示時はリンクテキストとして**対象ファイルのタイトル**（frontmatterの`title`）を表示する。
- **R-2**: wikilinkの対象は辞書コレクションのみとする（「wikilinkで書かれている = 辞書コンテンツ」）。辞書以外へのリンクは通常のMarkdownリンクで書く。
- **R-3**: 変換結果は通常の `<a href="/dict/[slug]">` として出力し、プレビュー・サイドペイン表示はクライアントJSで拡張する（JS無効環境では通常のページ遷移として機能する）。
- **R-4**: 辞書リンクは通常リンクと異なる特別デザインとする（本アイコン+アクセント色+点線下線。詳細は [`ui-design-spec.md`](../ui-design/ui-design-spec.md) の「辞書リンク」）。

### 適用範囲

- **R-5**: 辞書・記事・本（index・章）のすべての詳細ページ本文で、辞書リンクにホバープレビューとサイドペイン表示を適用する。
- **R-6**: サイドペイン内に表示された辞書本文中の辞書リンクも、ホバープレビューの対象とする。

### ホバープレビュー（デスクトップのみ）

- **R-7**: 辞書リンクにマウスホバーすると、プレビュー小窓に辞書の**本文全文**を表示する。小窓はリンク下に表示し、画面下端にかかる場合は上に反転する。
- **R-8**: プレビュー小窓内の辞書リンクは、さらにプレビューしない。クリック時は通常のページ遷移とする。
- **R-9**: モバイル（タッチ環境）ではプレビューを表示しない。

### サイドペイン（辞書表示欄）

- **R-10**: 詳細ページの右側に辞書表示欄を置く。デフォルトでは「辞書リンクを押すと、ここに内容が表示されます。」のようなメッセージを表示する。
- **R-11**: 本文の辞書リンクをクリックすると、画面遷移せずサイドペインにその辞書の内容（種別ラベル・タイトル・タグ・本文・辞書ページへのリンク）を表示する。
- **R-12**: サイドペイン内の辞書リンクをクリックすると、ペインの内容がその辞書に切り替わる。
- **R-13**: サイドペインは単体で履歴を管理し、進む/戻るができる。履歴は開いているブラウザページ単体で管理し、ページ遷移でリセットする。ブラウザの履歴（URL）には反映しない（リロードでリセットされてよい）。
- **R-14**: モバイル（lg未満）ではサイドペインの代わりに、フローティングボタン押下または辞書リンククリックでボトムシートを開いて表示する。

### embedパーシャル

- **R-15**: プレビュー・サイドペイン用のコンテンツは、Astroのページパーシャル（`export const partial = true`）で本文のみのHTML断片をビルド時に静的生成し、クライアントJSでfetchして挿入する（Markdownのレンダリング結果をそのまま再利用する）。
- **R-16**: fetchに失敗した場合はプレビュー/ペイン表示をあきらめ、リンクとしての機能（ページ遷移）は損なわない。

### 使用されている辞書一覧

- **R-17**: 詳細ページの右上（モバイルでは本文下部）に、本文でリンクされている辞書ページの一覧を小さく表示する。

### 辞書の逆リンク

- **R-18**: 辞書詳細ページの本文下に、その辞書がwikilinkでリンクされている公開コンテンツ（辞書・記事・章）の一覧を種別バッジ付きで表示する。
- **R-19**: 非公開コンテンツからのリンクは逆リンク一覧に載せない（本番ビルド。[`content-model.md`](content-model.md) R-11）。

## 3. データ定義

- embedパーシャルのURL: `/dict/[slug]/embed/`
  - 内容: 当該辞書の本文レンダリング結果のHTML断片（`<html>`/`<head>`等を含まない）と、ペイン表示に必要なメタ情報（タイトル・タグ）。詳細構造は実装時に [`../architecture.md`](../architecture.md) で定義。
- 辞書リンクの識別: 変換時に付与するdata属性（例: `data-dict-link` と `data-slug`）でクライアントJSが対象を特定する。属性名の確定は [`../markdown-pipeline/wikilink.md`](../markdown-pipeline/wikilink.md) 側。

## 4. 受入基準

- **AC-1**: 記事本文の `[[ownership]]` が `<a href="/dict/ownership">` +辞書タイトルのリンクとして出力され、特別デザインが適用される。（R-1〜R-4）
- **AC-2**: JSを無効にした状態で辞書リンクをクリックすると `/dict/[slug]` へ通常遷移する。（R-3）
- **AC-3**: デスクトップで辞書リンクにホバーすると本文全文のプレビュー小窓が表示され、小窓内の辞書リンクにホバーしてもプレビューは出ない。（R-7, R-8）
- **AC-4**: 辞書リンクをクリックしてもURLが変わらず、サイドペインに辞書内容が表示される。（R-11, R-13）
- **AC-5**: サイドペインで辞書A→Bと開き、戻るボタンでAに戻り、進むボタンでBに戻れる。ページをリロードすると履歴はリセットされデフォルト表示になる。（R-12, R-13）
- **AC-6**: `/dict/[slug]/embed/` がビルドで生成され、本文のみのHTML断片を返す。非公開辞書のembedは本番ビルドで生成されない。（R-15）
- **AC-7**: 辞書詳細の逆リンク一覧に、その辞書をリンクしている公開記事・公開章が表示され、非公開記事は表示されない。（R-18, R-19）
- **AC-8**: 詳細ページに使用されている辞書一覧が表示され、本文中のwikilink先と一致する。（R-17）

## 5. 未確定事項

- ホバープレビューの表示遅延（ホバー後何msで表示/非表示するか）は実装時に調整する。
- 使用されている辞書一覧のクリック挙動（ページ遷移 or サイドペイン表示）は実装時に決める。
