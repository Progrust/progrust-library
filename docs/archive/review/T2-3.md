## 指摘一覧

### 推奨

- **R-1: fetchにタイムアウトがなく、応答しないサーバがビルドを無期限に停止させうる**（`plugins/link-card.mjs:47`）— `fetch(url, { redirect: "follow" })` にタイムアウトがない。接続拒否やDNS失敗は即座にthrowしてフォールバックするが、**応答を返さずコネクションを保持するサーバ**に当たるとvisitorがawaitのまま進まず、ローカル/CIのビルドが失敗ではなくハングする（R-4「fetch失敗はビルドエラーにしない」の趣旨はフォールバックで満たしているが、ハングはフォールバック経路にすら入らない）。`fetch(url, { redirect: "follow", signal: AbortSignal.timeout(10_000) })` の1行で対処でき、タイムアウトはAbortErrorとしてthrowされるので既存のcatch→フォールバックがそのまま効く。本タスクで直すか、少なくとも link-card.md の「制約・残課題」に追記しておくこと（現状の残課題リストは正規化・文字コード・巨大HTML等を挙げているがタイムアウトは漏れている）

### 軽微

- **R-2: 取得したtitle/descriptionのHTML実体参照が二重エスケープされる**（`plugins/link-card.mjs:52-64`）— `<title>Foo &amp; Bar</title>` のようにソースHTML側でエスケープ済みの文字列をそのまま `escapeHtml` に通すため、出力が `Foo &amp;amp; Bar`（表示上 `Foo &amp; Bar`）になる。コミット済みキャッシュの実データ4件では非発生だが、`&`を含むタイトルのサイトで表示が崩れる。OGPパーサ強化（残課題に記載済みの「`<meta>`属性バリエーション等」）の際に実体参照デコードを合わせて入れるとよい
- **R-3: スキーム判定が小文字限定**（`plugins/link-card.mjs:146`）— `/^https?:\/\//` は `HTTPS://EXAMPLE.COM/` のような大文字スキーム（URLとして有効）にマッチせずカード化されない。安全側（fetchしない方向）への逸脱であり、rule.mdの記法例も小文字のため実害はほぼないが、意図的な制限ならガードのコメントに一言あると次の読者が迷わない
- **R-4: og:content内の連続改行でHTMLブロックが分断されうる**（`plugins/link-card.mjs:30-38`）— `extractMeta` の `content=["']([^"']*)["']` は改行を含む値を許容し、値に空行（連続改行）が含まれるとrawHtml再パース時にHTMLブロック（type 6）が空行で終了してカードHTMLの後半が生markdown扱いになる。極端なエッジケースであり、残課題「巨大HTML等のエッジケースは本番で強化」の範疇として軽微とする（対策する場合はnonEmptyでの改行→空白正規化で足りる）
