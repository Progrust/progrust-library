## 指摘一覧

### 要修正

なし。

### 推奨

1. **0件ヒット時に空のドロップダウン枠が表示される** — `src/scripts/search-box.ts:88-90`（`renderResults`）
   フィルタ結果が0件でも `results.classList.remove("hidden")` を実行するため、ボーダー + `py-1` だけの空箱が入力欄の下に出る。spec はドロップダウンの0件時挙動を定めていない（R-10 は一覧ページ絞込のみ）が、空枠の表示は意図した見た目とは考えにくい。0件時は隠す（または「該当なし」を出すなら spec §5 に決定を記録）ことを推奨。
2. **`KindBadge.astro` の複製注記が `search-box.ts` を含んでいない** — `src/components/KindBadge.astro:6-7`
   バッジデザイン変更時の同期先として `dict-pane.ts` の `DICT_BADGE_CLASS` のみを注記しているが、本タスクで `search-box.ts` の `BADGE_CLASS` にも同系クラスが複製された。注記に追記しないと、将来のバッジ変更時に検索ドロップダウンだけ取り残されるリスクがある。
3. **AC-2 / AC-6 のブラウザ目視確認の記録がない** — `docs/plan/phase-5-search.md`（T5-3 実施履歴「検証」）
   architecture §10 は UI を「ビルド成功 + 開発サーバでの目視」で確認するとしているが、実施履歴の検証記録はビルド成果物の静的確認（inline スクリプトの存在等）に留まる。AC-2（初期ロードで fetch されないこと・初回フォーカスで fetch されること）と AC-6（章クリックでの遷移）は実際のブラウザ操作でしか確認できないため、開発サーバでの目視結果を履歴に追記することを推奨。

### 軽微

1. **Tab キーでフォーカスを外してもドロップダウンが閉じない** — `src/scripts/search-box.ts:135-143`
   閉じる経路が Escape とボックス外クリックのみのため、キーボード操作で `blur` した場合はドロップダウンが開いたまま残る。spec 外（キーボード操作は §5 で初期実装対象外と決定済み）なので現状可だが、T5-4 以降での改善候補。
2. **Escape で閉じた後、再フォーカスしても入力があるまでドロップダウンが再表示されない** — `src/scripts/search-box.ts`
   `focus` はロードのみで再描画しないため。挙動として不自然ではなく spec 外のため現状可。

## 総合判定

**承認**。完了条件（search.md AC-2 / AC-6）と DoD を満たしており、要修正の指摘はない。推奨指摘（0件時の空枠・複製注記・目視記録）は次タスク以降での対応で差し支えない。
