## 指摘一覧

### 要修正

なし。

### 推奨

1. **`urlFor` と種別ユニオンの重複**（`src/lib/feed.ts:37-49` / `src/lib/search-index.ts:36-51`）
   URL導出ロジックがほぼ同一の2実装で存在する（`FeedType` と `SearchType` も同じ4値ユニオンの重複）。feed.ts 側のコメントは「search-index.ts の urlFor と同一」と言うが、実装は chapter 分岐が微妙に異なり（split する/しない。結果URLは同じ）、URL規則を変えるとき2箇所の修正が必要になる。SSoT の観点から `content.ts` 等への共通化を推奨。実害はないため次回リファクタタスクでの対応でよい。

### 軽微

1. **feeds-meta §5「未確定事項」に確定事項が残る**（`docs/spec/feeds-meta.md:53`）
   「URLは `/rss.xml`、表記は『RSS』で確定」は確定した事実であり、§2 の RSS 要求仕様側に移して §5 から削除する方が章の意味（未確定事項）と整合する。他の確定済み項目と合わせて T6 完了時に整理すれば十分。
2. **`buildContentFeed` と `buildContentSearchIndex` の集約部が構造的に同型**（`src/lib/feed.ts:81-111`）
   `getPublic*` 4種を `Promise.all` して型別に map する形が search-index.ts と重複。推奨1の共通化と合わせて検討対象だが、現状の可読性は良好で許容範囲。
