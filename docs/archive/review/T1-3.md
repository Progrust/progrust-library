## 指摘一覧

要修正・推奨の指摘はなし。軽微のみ。

### 軽微

1. **frontmatter抽出の正規表現がCRLF・空frontmatterで不成立** — `plugins/dict-index.mjs:28` / `plugins/wikilink.mjs:23`
   `/^---\n([\s\S]*?)\n---/` はCRLF改行のmdや`---\n---`（空frontmatter）にマッチせず、その場合 `fm = {}` → `public: true` 扱いに静かにフォールバックする。非公開エントリがCRLFで保存されると公開非対称ルール（R-14）の違反を見逃しうる。現状コンテンツはローカル執筆のLFのみで実害なし。T1-4の検証パスでも同じパースを使う見込みのため、その際に改行を`\r?\n`で許容するか、パース失敗を明示的に扱うことを検討するとよい。
2. **`title`フォールバックが拡張子付きファイル名** — `plugins/dict-index.mjs:34`
   `title`欠落時のフォールバックが `entry.name`（`ownership.md`）で拡張子を含む。zodスキーマ（AC-1）で`title`必須のため実運用では到達しないが、到達した場合の表示が不自然。slug（拡張子除去済み）をフォールバックにする方が一貫する。
3. **辞書索引がconfig評価時の1回読みのため、devサーバ実行中の辞書追加が反映されない** — `astro.config.mjs:10`
   `loadDictIndex`はconfig評価時に1回だけ走るため、`astro dev`中に辞書mdを追加してもサーバ再起動まで索引に反映されない（新規辞書へのwikilinkがリンク切れthrow扱いになる）。設計上の合意事項（config時直読み）でありspec違反ではないが、執筆体験に関わるため、どこかの時点で既知の挙動としてdocs（markdown-notation/rule.md等の執筆者向け文書）に一言あると親切。
