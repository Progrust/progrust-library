### 推奨

- **R-1: pages.md R-12 と ui-design-spec §312 のタグ表示の不整合**（`docs/spec/pages.md:55`）
  R-12 は「詳細ページ共通（辞書・記事・**章**）」として「タイトル / **タグ一覧** / 更新日 / 本文」を表示すると定めるが、確定モック由来の ui-design-spec §312 は「章は**タグも表示しない**」と定めており、実装（`ChapterLayout.astro` のヘッダー）は後者に従っている。章 frontmatter にはタグが存在する（タグ詳細 R-19 の対象）ため、見た目の確定を仕様へ還流するルール（spec先行更新・SSoT）に照らし、pages.md R-12 に章の例外（タグ非表示）を明記すべき。実装は確定モックに一致しており修正不要、文書側の追従のみ。

### 軽微

- **M-1: `toTarget` の台帳経由ルックアップが冗長**（`src/pages/books/[book]/[chapter].astro:35-41`）
  前後章エントリを `ledger.find()` で引き直しているが、`ChapterLedgerItem` は `title` も保持しているため、見つけた item をそのまま `{ href, order, title }` に使えば `entry.data.title` との二重参照が消える。あるいは `chapterOrderLabel(entry)` + href 組み立てで ledger 非依存にもできる。動作に問題はない。
- **M-2: 非公開スキップのテストが prev 方向のみ**（`tests/lib/content.test.ts` の `[AC-4] 間に非公開章がある場合`）
  `chapterNav` のスキップ検証が「x/c から見た prev」のみで、next 方向のスキップ（x/a から見て x/hidden を飛ばし x/c）が未検証。ロジックは対称だが、AC-4 の担保を明示するなら next 側のケースを1本足すとよい。
- **M-3: `chapterOrderLabel` の連番欠落時 `"00"` 表示**（`src/lib/content.ts`）
  filePath 欠落・連番なしファイル名では `00` ラベルが表示に出る。content-model R-7 の命名規則が守られる限り到達しないが、命名規則違反をビルド時に検知する仕組みは現状ない（将来の執筆ミス検知として検討余地。今回のスコープ外）。
