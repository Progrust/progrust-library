## 指摘一覧

### 要修正

- **F-1** `src/pages/dict/index.astro:16-26` — ページタイトル節が確定モック `dict-index.html` と相違し、完了条件「dict-indexモックと目視一致」を満たさない。モックは h1 `text-4xl sm:text-5xl tracking-tighter leading-none` + アクセント色の「.」付き（`辞書<span class="text-accent">.</span>`）、件数表記は h1 の隣（`flex flex-wrap items-baseline gap-x-4`）に `25 entries / created_at desc`、その下にリード文（「Rustの用語を1語1ページで蔵書した辞書。…」）、eyebrowは `mb-3`。実装は h1 `text-2xl sm:text-3xl tracking-tight`・ドットなし・件数は `justify-between` で右端寄せ（`/ created_at desc` なし）・リード文なし・eyebrow `mb-2`。h1サイズ差とリード文の有無は目視で明確に判別できる相違。意図的にトップと揃えたのであれば、その決定を記録した上でモック側（またはui-design-spec）を更新してから実装するのが本プロジェクトの進め方（spec先行）。
- **F-2** `src/components/ListFilter.astro` — 絞込UIがモックと複数点で相違する。①チップにタグ名の `#` 接頭辞がない（モックは `#Rust基礎 11`、実装は `Rust基礎 11`。61行目付近）。②チップに `bg-card` がない（モックは `border-line bg-card`、実装は枠線のみで背景がページ地のまま）。③件数表示が `{total} / {total}`（モックはヒット数を `text-strong font-semibold` で強調し `25 / 25 entries` と単位付き。47-49行目）。④絞込セクションにモックの `pb-8 border-b border-line`（カードグリッドとの区切り罫線）がない。⑤超過タグ表示（`+ N tags`）が罫線付きスパンだが、モックはアクセント色テキストのボタン風（`text-accent hover:underline`・枠線なし）。⑥キーワード入力がモックは `h-10` / `sm:max-w-xs`、実装は `h-9` / `max-w-sm`。P5で動作を配線する際の土台になるコンポーネントなので、見た目は本タスクでモックに揃えておくべき。
- **F-3** `docs/architecture.md:106` — §6 の主要コンポーネント一覧（`ArticleCard`（本一覧と共用可）/ `TagChip` / `TagFilter`（絞込UI））が実装（`EntryCard` / `ListFilter`。チップは独立コンポーネントにせずListFilter内に内包）と一致していない。implementation-rules §4 は「ファイル名は architecture.md 6章のコンポーネント名と1:1」と定めており、architecture.md 自身も「実装が進んで構造が変わったら本書を更新する」と定める。実装名の方が実態に合っている（EntryCardは記事・本共用、チップ独立化は現状不要）ため、architecture.md §6 を実装に合わせて更新するのが妥当（コード変更は不要）。

### 推奨

- **R-1** `docs/plan/phase-3-pages.md:100` — P5絞込JS用の予約 data-* 属性（`data-list-filter` / `data-total` / `data-filter-keyword` / `data-filter-count` / `data-filter-tag`）が plan の実施履歴（申し送り）にのみ記載されている。implementation-rules §4 は「`data-*` 属性コントラクトは ui-design-spec.md を正とする」と定めており、ui-design-spec には現状 `data-title` / `data-tags` しか記録がない。planはアーカイブされる文書なので、P5着手前に ui-design-spec のタグチップ（絞込UI）節へ属性コントラクトとして反映しておくと、SSoTが保たれ P5 実装が迷わない。
- **R-2** `src/components/DictCard.astro:20-30` — モックの辞書カードはタイトルに `leading-snug`、タグ行に `mt-auto`（+`gap-x-2 gap-y-1`）を持ち、グリッドの行内でカード高さが揃った際にタグが下端に揃う。実装（`gap-3` のみ・`mt-auto` なし）ではタイトルが2行に折り返すカードと1行のカードが混在するとタグ位置が不揃いになり、モックと見え方が変わる。F-1/F-2と併せての修正を推奨。

### 軽微

- **M-1** `src/components/DictCard.astro:17` / `EntryCard.astro:23` — `data-tags` の `|` 区切りは、タグ名に `|` を含むfrontmatterが書かれた場合に破綻する（現状のcontent-modelにタグ文字の制約なし）。区切り文字自体はui-design-spec由来のため実装は正しい。P5の絞込実装時に考慮するか、content-model にタグ名の文字制約を1行追記しておくと安全。
- **M-2** `src/components/ListFilter.astro:57-66` — チップは `disabled` だが `hover:border-accent hover:text-accent` が残っており、静的シェル段階でも操作可能に見える。P5で動作が入れば解消されるため軽微（F-2対応時に気になるなら `disabled:` バリアントで抑制可能）。
- **M-3** `src/components/ListFilter.astro` — モックにある0件時メッセージ要素（`no-hit`「条件に一致する辞書はありません。」、`hidden` 付き）が静的シェルに含まれていない。search R-10 の動作はP5スコープなので実装時で問題ないが、マークアップの置き場（一覧側かListFilter側か）を申し送りに含めておくと確実。
