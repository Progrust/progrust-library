## 指摘一覧

### 要修正

- **F-1: 脚注のある本文で、非表示用見出し「Footnotes」が可視表示され目次にも混入する**
  - 該当: `src/styles/global.css`（`.sr-only` 未定義）/ `src/components/Toc.astro:15`（`footnote-label` を除外していない）
  - パイプラインは脚注セクションに `<h2 class="sr-only" id="footnote-label">Footnotes</h2>` を出力するが、`sr-only` は Tailwind のスキャン対象（`src/` 等）に現れないためビルドCSSに含まれず（`dist/_astro/*.css` で 0 件を確認）、`.prose h2` のスタイルで**英語見出し「Footnotes」が本文に可視表示される**。さらに `render()` の `headings` にも含まれるため、目次に「Footnotes」項目が出る（`dist/articles/rust-performance-tuning/index.html` で確認）。デザイン（ui-design-spec）にない表示であり、実コンテンツで脚注を使うと必ず発生する
  - 修正案: ① `global.css` に `.sr-only` 相当の定義を追加（または Tailwind の `@source` でパイプライン出力を対象化）し可視化を止める、② `Toc.astro` / `MobileNav.astro` のフィルタで `slug === "footnote-label"` を除外する。①②両方の対応を推奨

### 推奨

- **R-1: 目次の対象範囲が spec（R-13: h1〜h4）と実装（h2〜h4）で不一致**
  - 該当: `src/components/Toc.astro:15`（`h.depth >= 2 && h.depth <= 4`）/ `docs/spec/pages.md` R-13
  - pages.md R-13 は「対象はh1〜h4の4段階（実運用ではh2〜h4想定）」と規定するが、実装は h1 を**一律**除外している。コメントの理由付けは「body 先頭 h1 = タイトル重複は除外」だが、実際は文中の h1 もすべて落ちる。AC-7（h2〜h4 の階層表示）は満たすため実害は小さいが、「spec を先に更新してから実装する」原則に沿い、R-13 を「h2〜h4（本文の h1 はタイトル重複のため目次対象外）」へ更新するか、先頭以外の h1 を目次に含めるかのどちらかに揃えること

### 軽微

- **M-1: 目次対象見出しのフィルタ条件が2箇所に重複**
  - 該当: `src/components/Toc.astro:15` と `src/components/MobileNav.astro:14`（`depth >= 2 && depth <= 4` が二重定義）。範囲を変える際（R-1 や F-1 の除外追加）に片方だけ直す事故が起きやすい。フィルタを `src/lib/`（またはどちらか一方）に寄せると安全。F-1/R-1 対応時に合わせて解消するとよい
- **M-2: architecture.md §9 の「本文は `.prose` 配下にスコープする」と実際のスコープが部分的に不一致**
  - 該当: `docs/architecture.md` §9 / `src/styles/global.css`
  - `details`・`blockquote` 等は `.prose` 配下だが、`.wikilink`・`.message`・`.astro-code`・`.code-filename`・`.link-card*` はグローバルセレクタ。P4 の辞書ペイン内表示（`.prose` 外）を見据えた意図なら妥当だが、§9 の記述をその旨がわかる表現に精緻化しておくこと
- **M-3: 目次の現在地追従の decision が spec に未反映**
  - 該当: `docs/spec/pages.md` §5 未確定事項
  - 「実装するかは実装時に決める」の項目が、実装確定（ユーザー確定・phase 履歴に記録済み）後も未確定のまま残っている。§5 から削除し R-13 側へ一文追記するのが SSoT 的に正しい
- **M-4: タグリンク先 `/tags/[タグ名]` は T3-6 まで未生成**
  - 該当: `src/layouts/DetailLayout.astro:52`。計画順序（T3-6 でタグページ実装）どおりであり対応不要。`encodeURIComponent` でのエンコードと T3-6 の `getStaticPaths` が生成する実パスの一致は T3-6 実装時に確認すること（日本語タグ名で percent-encoding の齟齬が出やすい）
