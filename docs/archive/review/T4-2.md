## 指摘一覧

### 要修正

なし。

### 推奨

1. **連打時に表示と履歴が食い違いうる（競合）** — `src/scripts/dict-pane.ts:208`（`navigateTo`）
   辞書Aのfetch中（未キャッシュ・低速）に辞書B（キャッシュ済み）をクリックすると、Bの描画後にAのfetchが解決し、最後にクリックしたのはBなのに表示がAで上書きされ、履歴も `[..., B, A]` とクリック順と逆になる。世代トークン（クリックごとに採番し、解決時に最新のみ描画）で防げる。発生頻度は低いためT4-3（同じfetchを共有するプレビュー実装）と併せての対処でよい。
2. **shift+クリック（新規ウィンドウ）がペイン表示に奪われる** — `src/scripts/dict-pane.ts:233`
   修飾クリックの除外が `metaKey` / `ctrlKey` のみで、`shiftKey`（新規ウィンドウで開く）が `preventDefault` されブラウザ標準動作を損なう。除外条件に `event.shiftKey` を追加すべき。

### 軽微

3. **slugのURLエンコードが不統一** — `src/scripts/dict-pane.ts:75`（fetch）・`:212`（`location.assign`）は未エンコード、`selectedMarkup`（`:131`）は `encodeURIComponent` 済み。slugはmdファイル名由来で現状は安全だが、方針をどちらかに揃えるとよい。
4. **fetch失敗（null）が恒久キャッシュされる** — `src/scripts/dict-pane.ts:98`
   一時的なネットワークエラーでも以後そのslugは常に通常遷移フォールバックになる。R-16のフォールバックで機能は損なわれないため実害はほぼないが、null解決時はキャッシュから除去すると再試行できる。
5. **モジュール変数 `history` が `window.history` を陰蔽** — `src/scripts/dict-pane.ts:136`
   モジュールスコープなので動作は正しいが、ブラウザグローバルと同名で誤読しやすい。`paneHistory` 等への改名を検討。
6. **`DICT_BADGE_CLASS` のクラス文字列複製** — `src/scripts/dict-pane.ts:113`
   `KindBadge.astro` の辞書バリアントと同一クラスの複製（実施履歴に設計判断として記録済み・dict-pane側に出所コメントあり）。ただし `KindBadge.astro` 側には複製先への言及がなく、バッジデザイン変更時に乖離しうる。KindBadge側にも相互参照コメントがあると安全。

## 総合判定

**承認**（要修正なし。推奨2件・軽微4件は次タスク以降での対応で可。特に指摘1・2はT4-3のプレビュー実装で同じクリック委譲・fetch共有を触る際に併せて対処するのが効率的）。
